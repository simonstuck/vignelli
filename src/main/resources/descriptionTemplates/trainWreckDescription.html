What is a Train Wreck?

In short, a train wreck is a method call chain that navigates the structure of the program to retrieve data that is needed for a computation.

For example, consider the method call chain customer.getAddress().getZipCode() which asks the customer for its address, only to ask that for its zip code.

The problem is that not only does the current class now know about the customer, it also knows about its address and more specifically about the zip code of this address. This makes the design fragile. Should any of the classes change, this may affect the current class, even though it does not own any address or zip code directly.

Instead, methods should only ever tell their neighbour objects to perform a certain task, they should never ask for data in order to do the task themselves.
