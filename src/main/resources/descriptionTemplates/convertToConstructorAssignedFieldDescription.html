The singleton instance retrieval in ${thisClass} causes coupling between ${thisClass} and ${expressionClass} to be very high as ${thisClass} cannot be used without ${expressionClass}.

Soon, we will therefore inject the singleton instance into ${thisClass} on instance creation, i.e. as a constructor parameter. To move towards this goal, we first assign the singleton instance to a field, initialised in the constructor.

In the following step, ensure that you:

  • Select the constructor to be the initialisation place for the field.
  • Choose to replace all occurrences of ${expression} in the class with the field.
  • Declare the resulting field final (good practice)
