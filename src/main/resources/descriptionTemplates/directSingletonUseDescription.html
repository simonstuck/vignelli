Direct Use of a Singleton

You appear to be using a singleton directly in your code.

The singleton pattern should be used to restrict the number of instances a class can have instead of enabling easier retrieval of a specific object.

Using a singleton instance directly in another class by calling the Singleton.getInstance() instance retrieval method increases a strong coupling between the singleton and the class it is used in, i.e. the class using the singleton can only ever use this one specific instance.

This can be a particular problem during testing. For this reason, instance retrieval of a singleton — be it a legitimate usage of the pattern or not — should only ideally occur once in the entire code base and the instances can be passed into any constructor whose class is using the instance. Even better, to reduce coupling even further, Singleton should implement a generic interface which can be injected into other classes using it. This allows for even easier testing.
